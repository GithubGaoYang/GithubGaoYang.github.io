关于 Widget Extension

Widget Extension 是 iOS 14 新出的一种扩展形式，类似于 Today Extension。
不过，Today Extension 只能用于负一屏，而 Widget Extension 可以放置在屏幕任何区域。

对于 Widget Extension，Apple 做了很多开发限制。

首先是开发语言，必须使用 SwiftUI 开发。
但是，并不是所有的 SwiftUI 控件在小部件中都是可用的。例如，一些 UIViewRepresentable 或 NSViewRepresentable 包装的 UIKit 或 AppKit 视图，ObservedObject、List 等 SwiftUI 控件或属性，在小部件中都是被禁止使用的。

具体可使用的控件可参考官方说明

在小部件样式方面，Apple 也做了限制，每个小部件只有三种固定大小。
分别是：22（systemSmall）、24（systemMedium）、4*4（systemLarge）。

Widget 交互事件

Widget 只允许点击操作，不允许使用手势、滑动等操作。

默认情况下，点击桌面上的 Widget 会直接打开 App，当然，开发者也可以指定打开 App 时的路径，Widget 提供 widgetURL(_:) 和 Link 两种传递 URL 的方式。

如果小组件只有一个单击目标，可以使用 widgetURL(_:) 处理；对于有多个单击目标的 Widget，可以使用 Link 包裹视图

与 OC 项目交互

一些项目可能是 Objective-C 开发的（我现在做的项目就是），在此项目上开发 Widget 与在 Swift 项目上开发并无太大区别，只是在交互时需要桥接一下

做过 OC 与 Swift 混编的应该对桥接很熟悉了

没做过也不要紧，只需要在 OC 项目上新建一个 Swift 语言的类，Xcode 会自动提示创建桥接文件，点击 Create Bridging Header 创建即可

在 OC 里调用 Swift 方法时，只需要引入 项目名-Swift.h 即可，如项目名为 WidgetDemo，只需要 #import "WidgetDemo-Swift.h"，剩下的跟普通编程没差别

同样的，在 Swift 里调用 OC 方法时，只需要在自动创建的 项目名-Bridging-Header.h（如项目名为 WidgetDemo，有"WidgetDemo-Bridging-Header.h）里引入要在 Swift 中调用的 OC 类即可

常见问题

1. 设置了刷新策略，但一直没有触发刷新
检查刷新时间是否设置的过短，低于5分钟的刷新间隔是无效的；
只有在 Widget 显示时，刷新时间才是相对准确的，锁屏或未显示 Widget 时，系统可能会延后刷新时机。

2. 设置好控件后，界面显示了一个黄底红色禁止符号的图案
使用了 Widget 不支持的控件，如：继承于 UIViewRepresentable 的控件。

3. Widget 中是否能显示 GIF 图
由于 Widget 不支持动态刷新，且无法使用UIViewRepresentable 或 NSViewRepresentable 包装的 UIKit 或 AppKit 视图，所以 Widget 中无法播放 GIF 图。
如果使用系统组件（如UIImage）加载 GIF 会显示第一帧的静图（测试用例有限，不确定是否有显示其他帧的情况）
如果使用可播放 GIF 的三方库，会显示黄底红色禁止标识符的图片，表示控件不可用

4. OC(Objective-C)项目可以用 Widget 吗
可以。参考与 OC 项目交互模块说明
